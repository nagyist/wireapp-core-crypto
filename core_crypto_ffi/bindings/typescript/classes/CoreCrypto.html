joinByExternalCommit

  • joinByExternalCommit(publicGroupState: Uint8Array, configuration?: CustomConfiguration): Promise<ConversationInitBundle>

  • Allows to create an external commit to "apply" to join a group through its public group state.

    If the Delivery Service accepts the external commit, you have to mergePendingGroupFromExternalCommit in order to get back a functional MLS group. On the opposite, if it rejects it, you can either retry by just calling again joinByExternalCommit, no need to clearPendingGroupFromExternalCommit. If you want to abort the operation (too many retries or the user decided to abort), you can use clearPendingGroupFromExternalCommit in order not to bloat the user's storage but nothing bad can happen if you forget to except some storage space wasted.

    Returns

    see ConversationInitBundle

    Parameters

    • publicGroupState: Uint8Array

      a TLS encoded PublicGroupState fetched from the Delivery Service

    • configuration: CustomConfiguration = {}

      configuration of the MLS group

    Returns Promise<ConversationInitBundle>

    • Defined in CoreCrypto.ts:957

  • init(params: CoreCryptoParams): Promise<CoreCrypto>

  • This is your entrypoint to initialize CoreCrypto!

    Simple init

     
    const cc = await CoreCrypto.init({ databaseName: "test", key: "test", clientId: "test" });
    // Do the rest with `cc`

    Custom Entropy seed init & wasm file location

     
    // FYI, this is the IETF test vector #1
    const entropySeed = Uint32Array.from([
      0xade0b876, 0x903df1a0, 0xe56a5d40, 0x28bd8653,
      0xb819d2bd, 0x1aed8da0, 0xccef36a8, 0xc70d778b,
      0x7c5941da, 0x8d485751, 0x3fe02477, 0x374ad8b8,
      0xf4b8436a, 0x1ca11815, 0x69b687c3, 0x8665eeb2,
    ]);

    const wasmFilePath = "/long/complicated/path/on/webserver/whatever.wasm";

    const cc = await CoreCrypto.init({
